# CDK Digital Retailing: A Practitioner's Market Analysis

Interactive portfolio piece built for my application to CDK Global's Product Manager, Digital Retailing role (JR8576).

## What This Is

A single-page HTML dashboard analyzing CDK's DXP platform, competitive landscape, and dealer adoption friction from the perspective of someone who has actually used these systems on a showroom floor. Every section includes an "Operator's View" that connects product strategy to real dealership workflows I experienced as VP of Business Operations at a Mercedes-Benz dealership group.

## Why It Exists

Most PM candidates can research a product. Very few have run F&I workflows, processed out-of-state deals under pressure, and watched salespeople abandon tools in real time. This dashboard proves I've done the product research AND lived the user experience. It's the kind of artifact that turns a resume into a conversation.

## Key Data Sources

- **97% repeat steps / 70% blame verification**: Cox Automotive, Digitization of Car Buying Study, 2023
- **13% use 6-7 disconnected systems (doubled YoY)**: CDK Friction Points Study, 2025
- **58% of shoppers experienced issues (up from 47%)**: CDK Friction Points Study, 2026
- **NPS drops 13 points per additional hour**: CDK Friction Points Study, 2026
- **NPS +54 under 1 hour / -3 over 4 hours**: CDK Friction Points Study, 2026
- **$5B market projection by 2033**: Grand View Research (verify independently before citing verbally)
- **Roadster acquisition $360M (2021)**: Public reporting
- **AI adoption 28% → 39%**: CDK Friction Points Study, 2026

## Tech

Single self-contained `index.html`. Tailwind CSS via CDN, Chart.js via CDN, Inter font via Google Fonts. No build step. Deploys directly to GitHub Pages.

## Deployment

```
git add index.html
git commit -m "CDK Digital Retailing portfolio piece"
git push origin main
```

Enable GitHub Pages on `main` branch, root directory. Live at `https://james-bonaguro.github.io/{repo-name}/`
